## Stoa is a private organisation

`agentic-stoa` (the GitHub org and the Gitea mirror under `agentic-stoa/*`) and its repos (`hum`, `content-factory`, `stoa-blog`) are **private third-party**. The `frank` repo is public.

Names and tech stacks of those repos are already documented across this repo (cluster manifests, runbook, blog posts, plan/spec files) and that level of disclosure is accepted — scrubbing it now would make the docs inaccurate, which is worse than the leak.

**Do not volunteer additional detail** beyond what is already documented:
- No new descriptions of what each repo *does* beyond build-system identity (Node / Python / Hugo).
- No code snippets, function names, business logic, customer references, or domain models from those repos.
- No screenshots of the repos' contents or PR diffs.
- When adding new operational notes that touch agentic-stoa, prefer placeholders (`<repo>`, `<the workload repo>`) for new examples even though existing examples use the real names.

If something would clearly cross this line — when in doubt, ask first.
