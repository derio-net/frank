# n8n-01 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Deploy a single-user n8n workflow automation instance on gpu-1 with dedicated PostgreSQL, Authentik forward-auth, and Prometheus metrics.

**Architecture:** Raw K8s manifests for n8n + Bitnami Helm chart for PostgreSQL, both in namespace `n8n-01`. Authentik proxy provider gates access; n8n's built-in auth is bootstrapped via init container with hardcoded credentials. Exposed at `192.168.55.216` via Cilium L2 LoadBalancer.

**Tech Stack:** n8n 2.13.4, Bitnami PostgreSQL chart 14.1.10, Longhorn storage, Cilium L2 IPAM, Authentik forward-auth, SOPS/age encryption

**Spec:** `docs/superpowers/specs/2026-03-29--agents--n8n-01-design.md`

### Task 1: Create SOPS-encrypted Secret

**Files:**
- Create: `secrets/n8n-01/n8n-01-secrets.yaml`

This must be done first as both PostgreSQL and n8n depend on it.

- [ ] **Step 1: Generate passwords and encryption key**

```bash
# Generate random passwords (do NOT commit these plaintext values)
PG_ADMIN_PASS=$(openssl rand -base64 24)
PG_USER_PASS=$(openssl rand -base64 24)
N8N_ENC_KEY=$(openssl rand -hex 32)
echo "postgres-password: $PG_ADMIN_PASS"
echo "password: $PG_USER_PASS"
echo "encryption-key: $N8N_ENC_KEY"
```

- [ ] **Step 2: Create the plaintext secret YAML**

Create `secrets/n8n-01/n8n-01-secrets.yaml` with the generated values:

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: n8n-01-secrets
  namespace: n8n-01
type: Opaque
stringData:
  # Bitnami PostgreSQL admin password
  postgres-password: <PG_ADMIN_PASS from step 1>
  # Bitnami PostgreSQL n8n user password
  password: <PG_USER_PASS from step 1>
  # n8n credential encryption key (for recoverability if PVC is lost)
  encryption-key: <N8N_ENC_KEY from step 1>
```

- [ ] **Step 3: Encrypt with SOPS**

```bash
cd /Users/derio/Docs/projects/HOMELAB/frank-cluster
sops --encrypt --in-place secrets/n8n-01/n8n-01-secrets.yaml
```

Verify encryption:
```bash
head -5 secrets/n8n-01/n8n-01-secrets.yaml
```
Expected: `stringData` values replaced with `ENC[AES256_GCM,...]` blocks. The `.sops.yaml` config auto-selects the age key and encrypts `stringData`.

- [ ] **Step 4: Commit**


### Task 2: Create PostgreSQL ArgoCD App

**Files:**
- Create: `apps/n8n-01-postgresql/values.yaml`
- Create: `apps/root/templates/n8n-01-postgresql.yaml`

Reference files:
- `apps/infisical-postgresql/values.yaml` — Bitnami chart values pattern
- `apps/root/templates/infisical-postgresql.yaml` — Multi-source Application CR pattern

- [ ] **Step 1: Create Helm values**

Create `apps/n8n-01-postgresql/values.yaml`:

```yaml
# PostgreSQL for n8n-01 — standalone instance in n8n-01 namespace
# Matches the infisical-postgresql pattern (Bitnami chart + SOPS existingSecret)
fullnameOverride: "n8n-01-postgresql"

architecture: standalone

# Docker Hub mirror workaround (docker.io/bitnami/postgresql images unavailable)
image:
  registry: mirror.gcr.io
  repository: bitnamilegacy/postgresql

auth:
  username: n8n
  database: n8n
  # Passwords sourced from SOPS bootstrap secret.
  # Keys: postgres-password (admin), password (n8n user)
  existingSecret: n8n-01-secrets

primary:
  persistence:
    size: 5Gi
    storageClass: longhorn
```

- [ ] **Step 2: Create Application CR**

Create `apps/root/templates/n8n-01-postgresql.yaml`:

```yaml
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: n8n-01-postgresql
  namespace: argocd
  finalizers:
    - resources-finalizer.argocd.argoproj.io
spec:
  project: infrastructure
  sources:
    - repoURL: registry-1.docker.io/bitnamicharts
      chart: postgresql
      targetRevision: "14.1.10"
      helm:
        releaseName: n8n-01-postgresql
        valueFiles:
          - $values/apps/n8n-01-postgresql/values.yaml
    - repoURL: {{ .Values.repoURL }}
      targetRevision: {{ .Values.targetRevision }}
      ref: values
  destination:
    server: {{ .Values.destination.server }}
    namespace: n8n-01
  syncPolicy:
    automated:
      prune: false
      selfHeal: true
    syncOptions:
      - ServerSideApply=true
      - CreateNamespace=false
      - RespectIgnoreDifferences=true
  ignoreDifferences:
    - group: ""
      kind: Secret
      jsonPointers:
        - /data
```

### Task 5: Create n8n-01 Deployment

**Files:**
- Create: `apps/n8n-01/manifests/deployment.yaml`

Reference: `apps/comfyui/manifests/deployment.yaml` — gpu-1 scheduling pattern

- [ ] **Step 1: Create deployment manifest**

Create `apps/n8n-01/manifests/deployment.yaml`:

```yaml
# n8n-01 workflow automation on gpu-1
# Single-user instance; auth gated by Authentik forward-auth
apiVersion: apps/v1
kind: Deployment
metadata:
  name: n8n-01
  namespace: n8n-01
  labels:
    app.kubernetes.io/name: n8n-01
    app.kubernetes.io/component: server
spec:
  replicas: 1
  selector:
    matchLabels:
      app.kubernetes.io/name: n8n-01
      app.kubernetes.io/component: server
  strategy:
    type: Recreate
  template:
    metadata:
      labels:
        app.kubernetes.io/name: n8n-01
        app.kubernetes.io/component: server
      annotations:
        prometheus.io/scrape: "true"
        prometheus.io/port: "5678"
        prometheus.io/path: "/metrics"
    spec:
      nodeSelector:
        kubernetes.io/hostname: gpu-1
      tolerations:
        - key: nvidia.com/gpu
          operator: Exists
          effect: NoSchedule
      securityContext:
        fsGroup: 1000
      terminationGracePeriodSeconds: 30
      initContainers:
        - name: bootstrap-admin
          image: docker.io/n8nio/n8n:2.13.4
          command:
            - /bin/sh
            - -c
            - |
              n8n user:create \
                --email admin@n8n.local \
                --firstName Admin \
                --lastName Admin \
                --password admin \
                --role global:owner \
              || true
          env:
            - name: DB_TYPE
              value: postgresdb
            - name: DB_POSTGRESDB_HOST
              value: n8n-01-postgresql
            - name: DB_POSTGRESDB_PORT
              value: "5432"
            - name: DB_POSTGRESDB_DATABASE
              value: n8n
            - name: DB_POSTGRESDB_USER
              value: n8n
            - name: DB_POSTGRESDB_PASSWORD
              valueFrom:
                secretKeyRef:
                  name: n8n-01-secrets
                  key: password
      containers:
        - name: n8n
          image: docker.io/n8nio/n8n:2.13.4
          ports:
            - name: http
              containerPort: 5678
              protocol: TCP
          env:
            - name: DB_TYPE
              value: postgresdb
            - name: DB_POSTGRESDB_HOST
              value: n8n-01-postgresql
            - name: DB_POSTGRESDB_PORT
              value: "5432"
            - name: DB_POSTGRESDB_DATABASE
              value: n8n
            - name: DB_POSTGRESDB_USER
              value: n8n
            - name: DB_POSTGRESDB_PASSWORD
              valueFrom:
                secretKeyRef:
                  name: n8n-01-secrets
                  key: password
            - name: N8N_ENCRYPTION_KEY
              valueFrom:
                secretKeyRef:
                  name: n8n-01-secrets
                  key: encryption-key
            - name: N8N_METRICS
              value: "true"
            - name: N8N_METRICS_PREFIX
              value: "n8n_"
            - name: N8N_HOST
              value: "0.0.0.0"
            - name: N8N_PORT
              value: "5678"
            - name: WEBHOOK_URL
              value: "https://n8n-01.frank.derio.net/"
          resources:
            requests:
              cpu: 500m
              memory: 512Mi
            limits:
              cpu: 2000m
              memory: 2Gi
          volumeMounts:
            - name: data
              mountPath: /home/node/.n8n
          startupProbe:
            httpGet:
              path: /healthz
              port: http
            periodSeconds: 10
            timeoutSeconds: 5
            failureThreshold: 30
          livenessProbe:
            httpGet:
              path: /healthz
              port: http
            periodSeconds: 10
            timeoutSeconds: 5
            failureThreshold: 3
          readinessProbe:
            httpGet:
              path: /healthz
              port: http
            periodSeconds: 5
            timeoutSeconds: 3
            failureThreshold: 3
      volumes:
        - name: data
          persistentVolumeClaim:
            claimName: n8n-01-data
```

**Implementation note:** The `--role global:owner` flag should be verified against `n8n user:create --help` in the 2.13.4 image before deployment. If the flag is not supported, remove it — the first user created is typically the owner by default. The `|| true` ensures the init container won't block either way.
